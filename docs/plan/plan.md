## 実装方針

- terrain-editor(C++20 / CMake / vcpkg、Win32 + Direct3D 12、Dear ImGui + imgui-node-editor)をコードベースの土台にする。ノードグラフ評価、キャッシュ、非同期評価、3D/2D プレビュー、プロジェクト保存の仕組みをそのまま流用する。
- 侵食ソルバは terrain-editor の `Multi-Scale Erosion`(Stream Power + Thermal + Deposition)を出発点にし、UVメトリック補正とワールドZポテンシャルを追加する。
- まず [uv-space-erosion.md](../uv-space-erosion.md) §8.1 の**簡易版**(歪みが小さい前提で UV グリッドを 2D ハイトフィールドとみなし、既存ソルバをほぼそのまま使う)で全体のパイプラインを通し、その後 §8.2 の**完全版**(メトリック補正入り自前ソルバ)へ進む。
- UVレイアウトはワールド比例UV(1テクセル = 4cm 固定)を強制する。これにより $s_u, s_v$ がカーブの $1 - \kappa v$ 項以外ほぼ 1 になり、CFL が均一、リルのピッチが全域で揃う。
- 道路〜路肩〜法面裾までを一枚の連続UVアトラスにする($v$ 方向に繋げる)。シェル分割は当面避ける。

## フェーズ

### Phase 0: プロジェクト基盤 ✅ (2026-07-23)

- [x] terrain-editor の取り込み方針を決める → **fork 方式**(ソース一式をコピー)。深い改変(新ノード・ソルバ拡張・UI変更)が前提のため、サブモジュールではなくコピーを採用。`src/`, `shaders/`, `assets/`, CMake 構成を取り込み、`data/`(生成物)と `docs/`(terrain-editor 固有)は対象外
- [x] ビルドが通る骨格を作る(CMake + vcpkg、`roadside_editor.exe`)
- [x] AGENTS.md / CLAUDE.md のビルド・検証記述を実態(C++ / CMake)に合わせて更新する

### Phase 1: リボン定義とメッシュ生成 ✅ (2026-07-23)

- [x] デモ用リボンソースノード(`Ribbon`): 縦断勾配 + 道路/路肩/法面プロファイル
- [x] センターライン入力: 既存 `Path` ノード(Catmull-Rom / 直線セグメント)を Ribbon の Path 入力ピンへ接続。エッジを端点から辿ってチェーン化し、弧長パラメータでリサンプリング。未接続時は直線デモセンターライン
- [x] スイーププロファイルからの曲線リボンメッシュ生成(`BuildRibbonWorldMesh`)
- [x] ワールド比例UVでのUV割り当て($u$ = 弧長基準、$v$ = 横方向オフセット、1テクセル = texelSizeCentimeters 固定)
- [x] 3D プレビューでの曲線リボン表示(Ribbon の World Preview 設定。CPU Mesh バックエンドのみ。カーブ内側は $0.95/\kappa$ で v をクランプしカスプ回避)
- [ ] (残) Path ポイントの高さ入力による縦断形状(現状 Absolute モードの点があるときのみ使用。それ以外は縦断勾配パラメータ)

### Phase 2: メトリックベイク

- [x] `P_z`、`N_z` のベイク(`HeightfieldGrid.baseZ / normalZ`。$N_z$ は $s_u = 1 - \kappa v$ 補正込みの差分から復元)
- [x] 曲率 $\kappa(u)$ の1D計算(`RibbonCenterline.curvature`。グリッドへの永続ベイクはソルバ補正実装時に検討)
- [ ] $s_u(u,v) = (1 - \kappa v)\,ds/du$、$s_v$、`dA` のベイク(曲率は1D配列で保持)
- [ ] 検証ツール: $\sqrt{EG - F^2}$ の色ベイク可視化、UV上の総面積×スケール補正 と実メッシュ総面積の比較(§8.3)

### Phase 3: 簡易版侵食(パイプライン貫通)

- [x] UVグリッドを 2D ハイトフィールドとみなして `Multi-Scale Erosion` を適用(2026-07-23)
- [x] ポテンシャルを $\phi = P_z + h \cdot N_z$ に置き換える(最重要。§4)— Ribbon ソースが heights に φ を焼き、ソルバは無改造で φ を侵食。h は $(\phi - P_z)/N_z$ で復元(`HeightfieldPreviewField::Displacement`)
- [ ] 隣接セル距離を実距離 $s_u\,du$ に置き換える(§3.3。直線デモでは $s_u = 1$ のため実質未対応)
- [ ] $s_u$ の下限クランプ(CFL対策。初期値 0.5 程度から実測で調整)
- [ ] displacement / mask テクスチャの書き出し(現状は Displacement ピンのマスク書き出し(8bit, 0.5 = 変位ゼロの正規化)のみ。メートル値の 16/32bit 書き出しは未対応)

### Phase 4: 完全版ソルバ

- [ ] メトリック補正入りの自前ソルバ(GPU compute 優先、CPU フォールバック)
- [ ] 境界フラックス: 法面裾から周囲地形(ワールドHF)への土砂吐き出しの転写
- [ ] 侵食結果の displacement を元メッシュに戻す際の法線再計算

### Phase 5: 仕上げ

- [ ] リル間隔とテクセル解像度の関係の実測、パラメータのプリセット化
- [ ] エクスポート形式の整備(DCC / ゲームエンジン向け)
