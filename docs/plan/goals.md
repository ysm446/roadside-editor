# Goals

作成日時: 2026-07-22 22:27
更新日時: 2026-07-22 22:27

## 目的

道路・路肩のようなリボン形状に対して、**UV空間上で hydraulic erosion を計算する** エディタを作る。ワールド空間ハイトフィールドでは実現できない解像度効率とヘアピン・立体交差への対応を、UV空間での侵食計算によって実現する。

技術コンセプトの詳細は [../uv-space-erosion.md](../uv-space-erosion.md) を参照。

## 完成形

次のワークフローが一通り回るツール:

1. センターライン(スプライン)を入力し、道路・路肩・法面のリボンメッシュをスイープ生成する
2. ワールド比例UV(例: 1テクセル = 4cm 固定)で連続UVアトラスに展開する
3. メトリック(`P`, `P_z`, `N_z`, `s_u`, `s_v`, `dA`)をベイクする
4. UV空間の2Dグリッド上で侵食シミュレーションを回す
   - ポテンシャルは $\phi = P_z + h \cdot N_z$(バンク・縦断勾配に沿って水が流れる)
   - メトリック補正により物理的に整合した流量・勾配を得る
5. 結果を displacement / mask テクスチャとして書き出す

UI・ノードグラフ・侵食ソルバは [terrain-editor](https://github.com/ysm446/terrain-editor)(ローカル: `C:\GitHub\terrain-editor`)を土台として再利用する。

## 重視する価値

- **計算効率**: UV空間での計算はワールドHFに対してセル数で2桁以上少ない。コリドー上にのみ高解像度を割り当てる。
- **リボン形状の数学的単純さの活用**: メトリックが対角($F = 0$)で、1Dの曲率配列だけで全テクセルのスケール補正が復元できる。この性質を崩さない設計を保つ。
- **物理的整合性**: 見た目だけでなく、片勾配で水が路肩へ集まる・縦断勾配に沿って流れるという挙動が自然に出ること。
- **既存資産の再利用**: terrain-editor のノードグラフ、Multi-Scale Erosion、GPU バックエンド、プレビューを流用し、本プロジェクト固有の実装(メトリック補正・ワールドZポテンシャル・境界フラックス)に集中する。

## 適用範囲の前提(スコープ外)

- 法面が垂直に近いケース($N_z \to 0$)は対象外
- 擁壁・切土の急崖は対象外
- 面はワールドZに対しておおむね高さ場であること
